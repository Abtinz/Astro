
# Astro AI Visualizer Suite

A multi-app workspace for generating AI-powered 3D visuals with Gemini:

- `floor-plan-3d`: convert 2D architectural floor plans into a rendered image and an interactive voxel-style 3D scene.
- `image-to-voxel-art`: generate or upload images, then transform them into downloadable voxel HTML scenes.

Both apps are built with React + TypeScript + Vite and use `@google/genai` for model calls.

## Project Overview

## Environment Variables

Both apps expect `GEMINI_API_KEY` from `.env.local`.

Create these files:

```bash
# for floor-plan-3d
cat > floor-plan-3d/.env.local <<'EOF'
GEMINI_API_KEY=your_real_key_here
EOF

# for image-to-voxel-art
cat > image-to-voxel-art/.env.local <<'EOF'
GEMINI_API_KEY=your_real_key_here
EOF
```

Important:

- Never commit `.env.local`.
- In this codebase, Vite maps `GEMINI_API_KEY` to `process.env.API_KEY` in app code.

## Quick Start (Run Both Apps)

Install dependencies:

```bash
(cd floor-plan-3d && npm install)
(cd image-to-voxel-art && npm install)
```

Run each app in separate terminals:

```bash
# Terminal 1
cd floor-plan-3d
npm run dev

# Terminal 2
cd image-to-voxel-art
npm run dev
```

Default local URLs:

- `floor-plan-3d`: `http://localhost:3001`
- `image-to-voxel-art`: `http://localhost:3000`

## App 1: floor-plan-3d

Location: `floor-plan-3d/`

### What it does

1. Upload a 2D floor plan.
2. Generate a styled 3D render using Gemini image generation.
3. Generate an interactive voxel HTML scene from that render.
4. Download render image or scene HTML.

### User flow

- Step 1: Upload
- Step 2: 3D Render
- Step 3: 3D Scene

### Scripts

```bash
cd floor-plan-3d
npm run dev      # start dev server
npm run build    # production build
npm run preview  # preview production build
```

### Key implementation files

- `src/App.tsx`: state machine and user flow orchestration
- `src/services/gemini.ts`: Gemini calls for render + voxel generation
- `src/utils/html.ts`: HTML extraction and post-processing (camera tweaks, overlays, z-fighting fixes, environment injection)
- `src/components/*`: stepper, upload zone, viewer, loading overlay

## App 2: image-to-voxel-art

Location: `image-to-voxel-art/`

### What it does

- Generate an image from prompt (Gemini Flash image model), or upload your own image.
- Convert image into voxel-style Three.js HTML scene (Gemini Pro stream).
- Browse built-in examples and download image/scene outputs.

### Scripts

```bash
cd image-to-voxel-art
npm run dev      # start dev server
npm run build    # production build
npm run preview  # preview production build
```

### Key implementation files

- `App.tsx`: UI and generation flow
- `services/gemini.ts`: image generation + streamed voxel generation
- `utils/html.ts`: cleanup/extraction helpers for model output
- `examples/`: static sample HTML scenes

## How the Pipeline Works

### floor-plan-3d pipeline

- Input floor plan + bundled style image -> Gemini image model (`nano-banana-pro-preview` as currently configured) -> rendered image.
- Rendered image -> Gemini streaming model (`gemini-3-pro-preview`) -> HTML/Three.js scene.
- Generated HTML is post-processed before preview/download.

### image-to-voxel-art pipeline

- Prompt (or uploaded image) -> optional optimization prompt -> Gemini image model (`gemini-2.5-flash-image`).
- Resulting image -> Gemini streaming model (`gemini-3-pro-preview`) -> HTML voxel scene.

## Troubleshooting

### `process.env.API_KEY` is undefined

- Confirm `.env.local` exists in the app directory.
- Confirm key name is exactly `GEMINI_API_KEY`.
- Restart `npm run dev` after changing env files.

### Dependency or lockfile errors

```bash
(cd floor-plan-3d && rm -rf node_modules package-lock.json && npm install)
(cd image-to-voxel-art && rm -rf node_modules package-lock.json && npm install)
```

### Port already in use

- `floor-plan-3d` uses `3001`.
- `image-to-voxel-art` uses `3000`.
- Stop conflicting process or update `vite.config.ts`.

### Gemini request failures

- Validate API key and quota.
- Check network access.
- Retry with a simpler prompt/image.

## Security Notes

- API calls are currently made client-side.
- Do not expose production keys in public frontends.
- For production, move Gemini calls behind a backend service and apply auth/rate limiting.
